How it works

Three workspaces, one window.

DevGitCenter is organised around the three things you juggle every day — your repositories, their pull requests, and the apps you run locally.

Git Board

Discover, watch, fetch and clone your local repos. See branch, ahead/behind and dirty status at a glance — then stage, diff, commit, push, pull and browse history in a GitHub Desktop-style Changes view.

Pull Requests

Pull open PRs from every watched repo across GitHub and Azure DevOps into one searchable list — with review state, comment counts and additions/deletions. One click deep-links to the provider for review.

App Center

Build, run, stop and restart your local apps — each on its own port — and watch their logs stream live. Framework presets, static folder serving and an OpenAPI mock get any project up in seconds.

Features

Everything your dev machine needs.

Native speed, a quiet premium UI, and the plumbing handled for you — from secure tokens to signed updates.

Repo status at a glance

Branch, ahead/behind counts, and uncommitted changes are surfaced on every row. Tag and group repos to keep big machines tidy.

Commit like a pro

A GitHub Desktop-style Changes page with a file tree, hunk-level diffs, selective staging, and a three-pane history browser — plus push, pull and sync.

Framework presets

One-click build & run for .NET, Node, React, Next, Angular, Vue and Express — or serve a static folder, or stand up a Swagger / OpenAPI API mock.

Live streamed logs

Every running app streams stdout/stderr straight into the UI, so you can spot a crash or a port clash the moment it happens. Start All / Stop All from the system tray.

Secure by default

Provider tokens are stored in your OS keychain — never in plain text. Repos, settings and app state live locally in SQLite on your machine.

Signed auto-updates

New versions are minisign-signed in CI and delivered straight from GitHub Releases. DevGitCenter verifies the signature, installs, and relaunches — no installers to chase.

Native & local-first

A real desktop app, not a browser tab.

DevGitCenter is built in Rust with Tauri, rendering through your OS WebView — so there's no bundled Chromium, the install is small, and it starts fast. Your data stays on your machine.

  • Tiny footprint — uses WebView2 / WKWebView / WebKitGTK
  • Repos & apps work fully offline; tokens only for PRs
  • Polished light & dark themes, keyboard-friendly UI

FAQ

Frequently asked questions

What is DevGitCenter for?

It's a single desktop dashboard for your local development: track and operate your Git repositories, watch their pull requests across GitHub and Azure DevOps, and build / run / monitor your local apps with live logs — without juggling a terminal, a browser and three other tools.

Which platforms are supported?

A signed installer is published for every release on Windows (x64 .exe), macOS (Apple Silicon & Intel .dmg) and Linux (.AppImage & .deb) — each with built-in, minisign-signed auto-updates.

Where is my data stored?

Entirely on your machine. Repos, accounts and app definitions live in a local SQLite database in your app-data folder, and provider access tokens are kept in your operating system's keychain / Credential Manager.

Do I need a GitHub or Azure account?

Only for pull requests. Add a personal access token to pull PRs from GitHub or Azure DevOps. Everything else — tracking repos, committing, and running local apps — works without any account.

How do updates work?

DevGitCenter checks for updates on launch and from Settings → Check for updates. Releases are minisign-signed in CI; the app verifies the signature before installing and relaunching. See the auto-update setup guide.

How do I report a bug or request a feature?

Open an issue on GitHub. Contributions are welcome.
